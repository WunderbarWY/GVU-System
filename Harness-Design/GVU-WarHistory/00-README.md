## 各文件内容简介

### INIT.md
项目总体背景、技术栈、关键文件路径、已确认 bug 清单、执行规则、BOOT 启动序列。

### 00-README.md（本文档）
文档集导航与快速入口。

### 01-PRD.md
产品需求文档：
- 功能需求
- 用户故事
- 验收标准
- 非功能性需求（性能、可靠性、离线支持）

### 03-平台适配决策树.md
用 🟢/🟡/🔴/⏳ 符号标注对现有代码的取舍：
- 🟢 保留
- 🟡 适配改造
- 🔴 移除
- ⏳ 未来增强但当前需预留设计

### 05-执行规划.md
最详细的实施手册：
- 每个文件的修改范围
- 需要新增/修改/删除的函数
- 代码级指令与伪代码
- 字段映射与预期行为

### 09-执行时序与里程碑.md
从 Phase -1 到 Phase 7 的时间线：
- 每阶段目标
- 预计耗时

### 10-AI自检清单.md
每个阶段的验收检查项，执行者逐条勾选。

### TODO.md

### 99-执行偏差日志.md
空白模板，用于记录执行中遇到的偏差：预期 vs 实际、原因、解决方案。

## 如何开始执行

1. 确认项目根目录存在且可写：
   ```bash
   ls /Users/waiyuen/Documents/Codex/2026-05-29/kimi-kimi-linear-api-linear-linear/
   ```
2. 打开 `app.js`、`src/api/supabase.js`、`supabase/migrations/20260617000000_initial_schema.sql` 进行快速通读。
3. 按 `09-执行时序与里程碑.md` 的 Phase -1 检查环境。
4. 进入 `TODO.md`，将 Phase -1 标记为进行中。
5. 按阶段推进，每完成一项更新 `TODO.md`，有偏差则记录到 `99-执行偏差日志.md`。

## 注意事项

- 所有路径均以项目根目录为基准。
- 不要一次性改动过多文件；按阶段推进，便于回滚。
- 遇到未在文档中覆盖的问题，优先按“最小改动原则”解决，并记录到 `99-执行偏差日志.md`。
